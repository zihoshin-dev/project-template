# 디자인 원칙 (Design Principles)

> **적용 범위**: dev_test 하위 모든 프로젝트
> **참조**: TDS (Toss Design System), Pencil.dev MCP
> **플랫폼**: Mobile, Web, PC

---

## 1. 핵심 철학

### 단순함과 속도
- 간단한 툴로 쉽고 빠르게 디자인
- TDS 컴포넌트 활용으로 3-5배 빠른 개발
- 커스텀 UI보다 디자인 시스템 우선

### 일관된 경험
- 사용자에게 **예측 가능하고 편리한** 경험 제공
- 플랫폼(모바일/웹/PC) 간 일관된 패턴 유지
- 창의성 제한이 아닌 품질 보장

---

## 2. 플랫폼별 가이드

### 2.1 Mobile/App (TDS 기준)

#### 해상도
| 방향 | 권장 범위 |
|------|----------|
| Portrait | 360×640 ~ 420×740 |
| Landscape | 640×360 ~ 740×420 |

#### 에셋 해상도
| 배율 | 용도 |
|------|------|
| 1x | 기본 해상도 |
| 2x | 고해상도 디바이스 (필수) |
| 3x | 그래픽 품질 중요 시에만 |

> ⚠️ 대부분 1x + 2x로 충분. 과도한 에셋 분할은 메모리/로딩 지연 유발

#### 전체화면 요구사항
- 콘텐츠가 화면을 완전히 채울 것
- 웹뷰 마진, 반투명 영역 금지
- 회전 시 비율 왜곡/레터박싱 금지
- Safe Area (노치, 다이나믹 아일랜드) 적절히 처리

---

### 2.2 Web/PC (TDS 확장)

#### 해상도
| 카테고리 | 권장 범위 |
|----------|----------|
| Mobile Web | 360px ~ 428px |
| Tablet | 768px ~ 1024px |
| Desktop | 1280px ~ 1920px |

#### 반응형 브레이크포인트
```css
/* Mobile First */
@media (min-width: 768px)  { /* Tablet */ }
@media (min-width: 1024px) { /* Desktop */ }
@media (min-width: 1440px) { /* Large Desktop */ }
```

#### 레이아웃 원칙
| 원칙 | 설명 |
|------|------|
| 최대 너비 | 콘텐츠 영역 1200px 이하 |
| 여백 | 모바일 16px, 데스크탑 24-32px |
| 그리드 | 12 컬럼 그리드 시스템 |
| 터치 타겟 | 최소 44x44px (웹에서도 유지) |

## 3. 컴포넌트 가이드 (TDS 기반)

### 필수 11개 컴포넌트

| 컴포넌트 | 용도 | PC 확장 |
|----------|------|---------|
| **Badge** | 상태/카테고리 표시 | 동일 |
| **Border** | 시각적 구분 | 동일 |
| **BottomCTA** | 하단 고정 CTA | 우측 고정 또는 인라인 |
| **Button** | 인터랙션 트리거 | hover 상태 추가 |
| **Asset** | 공통 그래픽 리소스 | SVG 우선 |
| **ListRow** | 리스트 아이템 | 테이블 레이아웃 가능 |
| **ListHeader** | 섹션 헤더 | 동일 |
| **Navigation** | 화면 네비게이션 | 사이드바/탑바 확장 |
| **Paragraph** | 텍스트 콘텐츠 | 줄 길이 60-80자 제한 |
| **Tab** | 다중 섹션 네비게이션 | 동일 |
| **Top** | 상단 인터페이스 | 동일 |

## 5. UX Writing 원칙

### 핵심 규칙

| 규칙 | 설명 | 예시 |
|------|------|------|
| **해요체** | 모든 문장 해요체 통일 | "~합니다" ❌ → "~해요" ✅ |
| **능동태** | 수동태 대신 능동태 | "처리되었습니다" ❌ → "처리했어요" ✅ |
| **긍정형** | 부정 대신 긍정 표현 | "안 돼요" ❌ → "~하면 할 수 있어요" ✅ |
| **캐주얼 경어** | 과도한 경어 자제 | "~시겠어요?" ❌ → "~할까요?" ✅ |

### 피해야 할 표현

| 피해야 할 표현 | 권장 표현 |
|---------------|----------|
| 본 서비스 | 이 서비스 |
| 당사 | 저희 |
| ~하십시오 | ~하세요, ~해요 |
| ~되었습니다 | ~했어요 |
| 계시다 | 있다 |
| 여쭤볼게요 | 확인할게요 |

### 예외 허용 (수동태)
- 서비스 종료/만료 안내
- 사용자 영향 설명
- 개인정보 관련 안심 문구

## 6. 다크 패턴 방지 (필수)

### 5대 금지 패턴

| # | 패턴 | 설명 |
|---|------|------|
| 1 | **진입 시 전체화면 시트** | 서비스 진입 시 광고/알림 동의 시트 금지 |
| 2 | **뒤로가기 가로채기** | 뒤로가기 시 알림 동의 요청 금지 |
| 3 | **강제 행동 (탈출구 없음)** | 거부 옵션 없는 CTA 금지 |
| 4 | **예상치 못한 전면 광고** | 메뉴 선택 중 갑작스러운 광고 금지 |
| 5 | **모호한 CTA 버튼** | 다음 행동이 불명확한 버튼 금지 |

## 7. 접근성 (A11y)

### 4대 원칙

| 원칙 | 설명 |
|------|------|
| **명확한 구조** | 시맨틱 HTML 사용 |
| **정확한 의미** | 적절한 레이블과 역할 부여 |
| **예측 가능한 상호작용** | 사용자 기대대로 동작 |
| **시각 정보 보완** | 이미지/아이콘 대체 텍스트 |

### 실무 가이드

```html
<!-- ❌ WRONG -->
<div onclick="...">클릭</div>
<button><button>중첩</button></button>

<!-- ✅ CORRECT -->
<button type="button" aria-label="메뉴 열기">
  <Icon name="menu" />
</button>
```

### 체크리스트
- [ ] 버튼 중첩 금지
- [ ] 테이블 row에 직접 onClick 금지
- [ ] 모든 인터랙티브 요소에 이름 부여
- [ ] 동일 이름 요소에 설명 추가
- [ ] input은 form 내부에 배치
- [ ] 이미지/아이콘에 alt 텍스트

## 8. Pencil.dev MCP 활용

### 언제 사용하는가
- `.pen` 파일 디자인 작업
- 빠른 UI 프로토타이핑
- 디자인 시스템 기반 화면 구성

### 주요 도구

| 도구 | 용도 |
|------|------|
| `get_editor_state()` | 현재 에디터 상태 확인 |
| `get_guidelines(topic)` | 디자인 가이드라인 조회 |
| `get_style_guide(tags)` | 스타일 가이드 조회 |
| `batch_design(operations)` | 디자인 작업 실행 |
| `get_screenshot()` | 디자인 검증 |

### 워크플로
```
1. get_editor_state() → 현재 상태 확인
2. get_guidelines('tailwind') → 가이드라인 로드
3. get_style_guide(tags) → 스타일 참조
4. batch_design(operations) → 디자인 실행
5. get_screenshot() → 결과 검증
```

> ⚠️ `.pen` 파일은 **Pencil MCP 도구로만** 읽기/쓰기 (Read/Grep 금지)
